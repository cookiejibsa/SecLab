개요 & 기초 문법
nmap의 기본 구조와 대상 지정 방법
기본 문법
syntax
nmap [옵션] [대상] # 단순 스캔 nmap 192.168.1.1 # 옵션 조합 nmap -sS -O -T4 192.168.1.0/24 # 포트 + 버전 + 스크립트 nmap -p 80,443 -sV -sC example.com
대상(Target) 지정 방법
표기법설명예시
192.168.1.1단일 호스트nmap 192.168.1.1
192.168.1.0/24CIDR 서브넷 (256개 호스트)nmap 192.168.1.0/24

--exclude 10.0.0.1특정 호스트 제외nmap 10.0.0.0/24 --exclude 10.0.0.1
--excludefile ex.txt파일 기반 제외 목록nmap 10.0.0.0/24 --excludefile ex.txt
포트 상태

closed — RST 응답
filtered — 방화벽 차단
unfiltered — 접근 가능, 상태 불명
open|filtered — 응답 없음

스캔 유형
TCP, UDP, 스텔스 스캔의 모든 방식
TCP 스캔
UDP / 기타
스텔스 스캔
비교표
옵션이름설명권한
-sS SYN (Half-open) SYN 패킷만 전송, RST로 닫음. 가장 빠르고 널리 사용됨. 로그에 기록되지 않는 경우 많음 root
-sT TCP Connect OS의 connect() 시스템 콜 사용. 완전한 3-way handshake. 로그에 남을 가능성 높음 일반
-sA TCP ACK 방화벽 규칙 파악에 사용. 포트 열림/닫힘이 아닌 필터링 여부를 탐지 root
-sW TCP Window ACK와 유사하지만 TCP Window 크기로 상태 판단. 일부 시스템에 효과적 root
-sM TCP Maimon FIN/ACK 조합. BSD 기반 시스템에서 필터된 포트 탐지 root
예시
# SYN 스캔 — 가장 일반적 sudo nmap -sS 192.168.1.1 # root 없이 TCP Connect 스캔 nmap -sT 192.168.1.1 # ACK 스캔으로 방화벽 매핑 sudo nmap -sA -p 1-1024 192.168.1.1
옵션이름설명
-sUUDP 스캔UDP 포트 스캔. 느리지만 DNS(53), SNMP(161), DHCP(67) 등 탐지에 필수. ICMP unreachable 응답으로 판단
-sYSCTP INITSCTP 프로토콜 스캔. 텔레콤 네트워크에서 사용
-sZSCTP COOKIE-ECHO상태 저장 방화벽 우회에 유용한 SCTP 스캔
-sOIP 프로토콜어떤 IP 프로토콜(TCP=6, UDP=17 등)이 지원되는지 탐지
-sI <zombie>Idle/Zombie완전한 익명 스캔. 좀비 호스트의 IP ID를 사이드채널로 활용
-b <ftp>FTP BounceFTP 서버를 프록시로 이용한 간접 스캔 (대부분 패치됨)
예시
# UDP + TCP SYN 동시 스캔 sudo nmap -sS -sU 192.168.1.1 # UDP 상위 100포트 빠른 스캔 sudo nmap -sU --top-ports 100 192.168.1.1 # DNS, SNMP, DHCP 포트 sudo nmap -sU -p 53,161,67 192.168.1.1
RFC 793 기반 스텔스 스캔은 Windows에서 동작하지 않습니다. Unix/Linux 대상에서만 유효합니다.
옵션플래그 조합설명
-sN플래그 없음TCP 플래그를 전혀 설정하지 않음. RFC 793 구현 시스템에서 closed 포트는 RST 응답, 방화벽 우회 가능
-sFFINFIN 플래그만 설정. 일부 시스템에서 stateful 방화벽 우회
-sXFIN+PSH+URG세 플래그를 모두 켠 "크리스마스트리" 패킷. 다채로운 패킷으로 일부 IDS 우회
--scanflags커스텀 플래그원하는 플래그 조합 직접 지정: URG, ACK, PSH, RST, SYN, FIN
예시
# Null 스캔 sudo nmap -sN 192.168.1.1 # Xmas 스캔 sudo nmap -sX 192.168.1.1 # 커스텀 플래그 (FIN+URG) sudo nmap --scanflags FINURG 192.168.1.1

호스트 발견
살아있는 호스트를 효율적으로 탐색하는 방법
호스트 발견 옵션
옵션설명
-sn포트 스캔 없이 호스트 발견만 (구 -sP)
-Pn호스트 발견 생략 — 모든 호스트를 온라인으로 간주하고 스캔
-sL목록 스캔 — 실제 스캔 없이 대상 목록만 출력 (DNS 조회 포함)
-PRARP Ping — 로컬 네트워크에서 가장 빠르고 신뢰도 높음
-PEICMP Echo 요청 (ping)
-PPICMP 타임스탬프 요청
-PMICMP 서브넷 마스크 요청
-PS <포트>TCP SYN Ping (지정 포트로, 기본 80)
-PA <포트>TCP ACK Ping
-PU <포트>UDP Ping
-PY <포트>SCTP INIT Ping
-nDNS 해석 비활성화 — 속도 향상에 큰 효과
-R모든 대상에 DNS 역방향 조회 강제
--dns-servers <IP>사용할 DNS 서버 직접 지정
상황별 예시
bash
# LAN에서 살아있는 호스트 발견 (ARP - 가장 빠름) sudo nmap -PR -sn 192.168.1.0/24 # ICMP가 차단된 경우 TCP로 핑 nmap -PS80,443 -sn 192.168.1.0/24 # 호스트 발견 건너뛰고 바로 포트 스캔 nmap -Pn 192.168.1.1 # DNS 없이 빠른 호스트 발견 nmap -sn -n 192.168.1.0/24

버전 / OS 탐지
서비스 버전과 운영체제를 식별하는 옵션
버전 탐지 옵션
옵션설명
-sV서비스/버전 탐지 활성화. 프로브를 전송해 서비스 이름과 버전 확인
--version-intensity <0-9>탐지 강도 설정. 0=가장 빠름(덜 정확), 9=가장 느림(가장 정확). 기본값 7
--version-light강도 2 — 빠른 버전 탐지 (정확도 낮음)
--version-all강도 9 — 모든 프로브 사용 (가장 느림, 가장 정확)
--version-trace버전 탐지 과정 디버그 출력
OS 탐지 옵션
옵션설명
-OOS 탐지 활성화. TCP/IP 스택 지문(fingerprint)으로 OS 추측. root 필요
--osscan-limit열린 포트와 닫힌 포트 모두 있는 경우에만 OS 탐지 시도 (성능 향상)
--osscan-guess확실하지 않은 경우 가장 가능성 높은 OS 추측해서 표시
--max-os-tries <n>OS 탐지 재시도 횟수 제한 (기본 5)
예시 및 출력
bash
# 버전 + OS 탐지 sudo nmap -sV -O 192.168.1.1 # 가장 정확한 버전 탐지 sudo nmap -sV --version-all 192.168.1.1 # 전체 종합 스캔 (-A = -sV -O -sC --traceroute) sudo nmap -A -T4 192.168.1.1 # 출력 예시: # 22/tcp open ssh OpenSSH 8.9p1 Ubuntu # 80/tcp open http Apache httpd 2.4.54 # 443/tcp open ssl/http nginx 1.22.1 # OS: Linux 5.15 - 5.19 (96%)

카테고리 — 클릭해서 자세히 보기
실전 예시
bash
# 기본 스크립트 + 버전 탐지 nmap -sC -sV 192.168.1.1 # 전체 취약점 스캔 sudo nmap --script=vuln 192.168.1.1 # SMB EternalBlue (MS17-010) 탐지 nmap --script=smb-vuln-ms17-010 -p 445 192.168.1.0/24 # HTTP 정보 수집 nmap --script=http-title,http-headers,http-methods -p 80,443 target # FTP 익명 로그인 탐지 nmap --script=ftp-anon -p 21 192.168.1.0/24 # SSH 브루트포스 (주의!) nmap --script=ssh-brute --script-args userdb=users.txt,passdb=pass.txt -p 22 target # SSL 취약점 (HeartBleed, POODLE) nmap --script=ssl-heartbleed,ssl-poodle,ssl-dh-params -p 443 target

방화벽 / IDS 우회
패킷 조작으로 탐지를 피하는 기술
이 기능들은 반드시 권한이 있는 시스템에서만 사용해야 합니다. 무단 스캔은 불법입니다.
패킷 조작 옵션
옵션설명
-fIP 단편화 — 패킷을 8바이트 조각으로 분할하여 DPI(심층 패킷 검사) 우회
-f -f더 작은 단편화 (16바이트). --mtu 옵션과 동일 효과
--mtu <크기>MTU 직접 지정 (8의 배수). 방화벽이 단편화된 패킷을 재조립하지 않으면 우회 가능
-D <decoy1,decoy2,...>디코이(위장 IP) 사용 — 로그에 여러 IP가 섞여 실제 스캐너 식별 어렵게 함
-D RND:1010개의 랜덤 디코이 자동 생성. ME로 자신의 IP 위치 지정 가능
-S <IP>소스 IP 주소 위조 (스푸핑). 응답은 수신 불가 — 무응답 스캔에만 의미
-e <인터페이스>특정 네트워크 인터페이스 사용 (eth0, wlan0 등)
-g <포트> / --source-port소스 포트 위장 — 80이나 53으로 설정하면 일부 방화벽 통과 가능
--proxies <URL,...>HTTP/SOCKS4 프록시 체인 경유 스캔
--data-length <길이>랜덤 데이터로 패킷 크기 조절 — 시그니처 기반 탐지 우회
--ip-options <옵션>IP 옵션 필드 설정 (레코드 라우트, 느슨한 소스 라우팅 등)
--ttl <값>IP TTL 필드 직접 설정
--spoof-mac <MAC>MAC 주소 위조. 0이면 랜덤, 공급업체 이름도 사용 가능
--badsum잘못된 체크섬으로 패킷 전송 — 방화벽/IDS 반응 여부 테스트
우회 기법 예시
bash

출력 & 저장
스캔 결과를 다양한 형식으로 저장하고 분석하는 방법
출력 형식 옵션
옵션형식설명
-oN <파일>Normal사람이 읽기 쉬운 텍스트 형식 (.nmap). 기본 터미널 출력과 동일
-oX <파일>XML구조화된 XML 형식. Metasploit, Nessus 등 보안 도구와 연동에 적합
-oG <파일>Grepablegrep/awk 처리 최적화. 한 호스트 당 한 줄 (.gnmap)
-oS <파일>Script Kiddie특수문자로 꾸민 농담성 형식 (실용적 용도 없음)
-oA <기본명>All (세 가지)Normal(.nmap) + XML(.xml) + Grepable(.gnmap) 동시 저장 — 추천
--append-output추가기존 파일에 결과 추가 (덮어쓰기 방지)
상세도 & 디버그 옵션
옵션설명
-v / -vv상세 출력 — 포트 발견 시 즉시 표시. -vv는 더욱 상세
-d / -d9디버그 레벨 (1-9). -d9는 최대 디버그
--reason각 포트 상태 판단 이유 표시 (어떤 패킷을 받았는지)
--open열린(open) 포트만 출력 — 결과 간소화에 유용
--packet-trace전송/수신된 모든 패킷 표시 (디버깅용)
--iflist네트워크 인터페이스 및 라우팅 테이블 목록 표시
--resume <파일>중단된 스캔을 grepable 또는 XML 파일로부터 재개
출력 활용 예시
bash
# 세 가지 형식 동시 저장 (가장 추천) nmap -sS -sV -oA scan_result 192.168.1.0/24 # Grepable 출력에서 열린 포트 추출 nmap -oG - 192.168.1.0/24 | grep 'open' # XML을 HTML 보고서로 변환 nmap -oX out.xml 192.168.1.1 xsltproc /usr/share/nmap/nmap.xsl out.xml > report.html # 열린 포트만, 이유 포함 nmap --open --reason 192.168.1.1 # 중단된 스캔 재개 nmap --resume scan_result.gnmap
